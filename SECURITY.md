# Security Policy

## Supported versions

Security fixes are applied to the latest released version.

## Reporting a vulnerability

Do not disclose a vulnerability or any secret in a public issue. Use the repository's **Security > Advisories > Report a vulnerability** flow. If private vulnerability reporting is unavailable, open a minimal issue asking the maintainer to establish private contact without including technical details.

Never include TryHackMe cookies, session values, API keys, or access tokens in a report, test fixture, screenshot, or log.
